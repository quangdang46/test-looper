## 1. Overview

Create `todo.py`, a single-file CLI tool that manages a flat to-do list stored in `TODO.md`. Tasks are represented as Markdown checklist items (`- [ ]` / `- [x]`) with a numeric ID and description. No external dependencies — only Python stdlib (`argparse`, `re`, `pathlib`).

## 3. CLI Interface

All operations are driven by a single `python todo.py` invocation with `argparse` flags. Only one action flag is processed per invocation (last specified wins if multiple are given).

| Usage | Description |
|---|---|
| `python todo.py` or `python todo.py --show` | Print all pending tasks, or "No tasks" if none. |
| `python todo.py --add "buy milk"` | Append a new pending task. |
| `python todo.py --done 2` | Mark task with ID `2` as done (`- [x]`). |
| `python todo.py --rm 3` | Remove task with ID `3` from the file entirely. |
| `python todo.py --clear` | Remove all `- [x]` (completed) lines from the file. |
| `python todo.py --help` | Print usage (auto-generated by argparse). |

### 3.1 Argument precedence

If multiple action flags are supplied, the tool processes **only the last** in this defined order (as determined by argparse positional ordering plus mutual exclusion logic). However, the simpler design is to check in a priority order and only handle once. For robustness: `--show` is the default; all other flags are mutually exclusive at the semantic level.

## 4. File Format (`TODO.md`)

```markdown
# Todo

- [ ] 1. buy milk
- [ ] 2. call mom
- [x] 3. finish homework
```

- **Header:** `# Todo` on its own line.
- **Blank line** between header and task list.
- **Each task** is a Markdown checklist item:
  - `- [ ] ` for pending
  - `- [x] ` for done
  - Followed by `<id>. <description>` where `id` is a positive integer.
- File encoding is UTF-8.

### 4.1 ID assignment

- Task IDs are sequential integers starting at **1**.
- When a new task is added, its ID = `max(existing IDs) + 1`.
- When a task is deleted, its ID is **never reused**.
- IDs of remaining tasks are **stable** — deleting task `2` does not renumber tasks `3` → `2`.

### 4.2 Whitespace / formatting tolerance when reading

- The reader strips leading/trailing whitespace from each line before matching.
- Lines that don't match `- [ ]` / `- [x]` are passed through verbatim (for comments, blank lines, etc.).
- The `# Todo` header is required at the top of the file for the tool to consider the file valid.

## 5. Operations — Detailed Behavior

### 5.1 Show (`--show` / default)

```
$ python todo.py
Tasks:
  - [ ] 1. buy milk
  - [ ] 2. call mom
  - [x] 3. finish homework
```

If no pending tasks exist: print `No tasks` (including the case where only done tasks exist, or the file is empty aside from the header).

### 5.2 Add (`--add <text>`)

Append a new task line to `TODO.md` just before the end-of-file. ID = max current ID + 1 (or 1 if no tasks exist).

```
$ python todo.py --add "wash dishes"
```

Result in `TODO.md`:
```
- [ ] 4. wash dishes
```

### 5.3 Done (`--done <id>`)

Replace `[ ]` with `[x]` on the matching task line.

```
$ python todo.py --done 2
```

Result: line `- [ ] 2. call mom` → `- [x] 2. call mom`

If `<id>` does not exist: print `Task <id> not found` to stderr and exit with code 1.

### 5.4 Remove (`--rm <id>`)

Delete the entire matching line from `TODO.md`. Other lines are untouched.

```
$ python todo.py --rm 3
```

Result: the line `- [x] 3. finish homework` is removed.

If `<id>` does not exist: print `Task <id> not found` to stderr and exit with code 1.

### 5.5 Clear (`--clear`)

Delete every `- [x] ...` line from the file. `- [ ]` lines and non-task lines are kept.

```
$ python todo.py --clear
```

If no done tasks exist: print `No completed tasks to clear.` to stdout.

## 6. Edge Cases

| Scenario | Behavior |
|---|---|
| `TODO.md` does not exist | Create it with `# Todo\n\n` header. Then proceed with the requested operation (e.g., `--add` still works). |
| `TODO.md` exists but has no `# Todo` header | Print `No tasks` (treat as empty). For `--done`/`--rm`: report `<id> not found`. For `--add`: insert header + task. For `--clear`: no-op. |
| `TODO.md` exists and has header but no tasks | Print `No tasks`. |
| Invalid task ID (non-integer, zero, negative) | argparse handles type=int natively. If the argument is missing or invalid, argparse prints its own error. |
| Task ID not found (`--done` / `--rm`) | Print `Task <id> not found` to stderr, exit 1. |
| Duplicate description added | Allowed — same text gets a new (higher) ID. |
| I/O error (permissions, disk full) | Print error message to stderr and exit with code 1. |
| Concurrent edits | Not handled — single-user, single-process use assumed. |

## 7. Implementation Plan

### Step 1: Create `todo.py`

A single Python file with these functions:

| Function | Purpose |
|---|---|
| `parse_args()` | Configure argparse, parse CLI flags. |
| `read_tasks(path)` | Parse `TODO.md`, return `(header_line, blank_line, list of task dicts)` and `list of non-task lines`. Each task dict: `{id: int, done: bool, desc: str, line: str}`. |
| `write_tasks(path, header, blank, tasks, non_task_lines)` | Serialize tasks + non-task lines back to `TODO.md`. |
| `show(tasks)` | Print task list or "No tasks". |
| `add_task(path, text)` | Read, append new pending task, write. |
| `mark_done(path, task_id)` | Read, toggle `[ ]` to `[x]`, write. |
| `remove_task(path, task_id)` | Read, delete matching line, write. |
| `clear_done(path)` | Read, delete all `- [x]` lines, write. |
| `main()` | Entry point — parse args, dispatch to operation. |

#### Regex pattern

```python
TASK_RE = re.compile(
    r'^- \[(?P<status>[ x])\] (?P<id>\d+)\.\s*(?P<desc>.+)$'
)
```

- `status`: `' '` or `'x'`
- `id`: integer (captured as string, cast to int)
- `desc`: rest of line after `<id>. `

Non-matching lines are preserved verbatim in their original order among the non-task lines.

#### File handling

- Use `pathlib.Path` for path operations.
- Read/write with `path.read_text(encoding='utf-8')` / `path.write_text(encoding='utf-8')`.
- On read failure (FileNotFoundError → treat as empty; PermissionError / OSError → print to stderr and exit 1).

## 10. Acceptance Criteria

1. `python todo.py --add "task"` creates `TODO.md` with header and one pending task.
2. `python todo.py` (no flags) prints pending tasks.
3. `python todo.py --done 1` marks task 1 as completed.
4. `python todo.py --rm 1` removes task 1.
5. `python todo.py --clear` removes all `[x]` tasks.
6. Invalid IDs produce `"Task <id> not found"` on stderr, exit code 1.
7. Missing `TODO.md` is auto-created.
8. Non-task lines (blank lines, comments) survive round-trips.
9. All operations survive empty or header-only files.
